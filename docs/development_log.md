# 개발 일지

## 2026-06-21

- 최종 주제를 Campus Convenience Finder에서 Where Is My Kimbap? 4.0으로 변경했다.
- 3.0 Java/MySQL/웹 프로젝트는 읽기 전용으로 분석하고 신규 폴더에 Python/Flet/DuckDB 구조를 만들었다.
- 10개 테이블의 DuckDB 스키마를 작성하고 빈 필수값, 음수 가격·재고, 절대 이미지 경로 제약을 검증했다.
- 100개 상품, 5개 매장, 316개 재고 관계를 가진 결정적 시드를 작성했다.
- 시드를 두 번 실행한 전후의 모든 테이블 건수가 동일함을 확인했다.
- 상품 5개는 재고 관계를 의도적으로 만들지 않아 products 기준 LEFT JOIN 동작을 검증했다.
- 사용자, 매장, 상품, 장바구니, 주문 Repository와 통합 Service를 구현했다.
- DuckDB의 FK 삭제 제한을 확인하여 장바구니 마스터를 유지하고 품목만 비우는 모델로 조정했다.
- 자동 테스트 8개가 모두 통과했다.
- Flet 0.85.2로 로그인, 주변 매장, 매장 상품, 상품 비교, 장바구니, 주문, 관리자 데이터 관리 화면을 구현했다.
- 상품·재고 생성/조회/수정/삭제와 이미지 fallback 테스트를 추가해 자동 테스트 11개가 통과했다.
- Flet 웹 서버를 포트 8765에서 실행하여 HTTP 200과 Flet HTML 응답을 확인했다.
- 내장 브라우저 연결 정책 오류로 자동 클릭 및 화면 캡처는 보류했다. 서버와 임시 로그는 검증 후 종료·삭제했다.
- Flet 자체 `Page.take_screenshot()`과 숨김 렌더러를 사용해 로그인 및 6개 기능 화면을 실제 PNG로 캡처했다.
- 상품·장바구니·주문 카드의 무한 높이 제약을 발견해 내부 Column에 폭을 지정하고 실제 화면과 캡처를 정상화했다.
- Crow's Feet ERD/ERD Editor JSON, Use Case, Sequence, Repository Interface, 전체 구성도, 구현 전 와이어프레임 6개를 생성했다.
- JOIN 3종의 전체 결과 CSV와 보고서용 PNG를 생성했고 LEFT JOIN 결과에서 재고 미등록 NULL 상품 5개를 확인했다.
